# /em:hard-call — Framework para Decisões Sem Boas Opções

**Comando:** `/em:hard-call <decision>`

Para as decisões que te mantêm acordado às 3 da manhã. Demitir um co-fundador. Dispensar 20% do time. Encerrar um produto que os clientes adoram. Pivotar. Fechar a empresa.

Essas decisões não têm uma resposta certa. Têm uma resposta menos errada. Este framework ajuda você a encontrá-la.

---

## Por Que Essas Decisões São Difíceis

Não porque os dados não são claros. Frequentemente, os dados são claros. São difíceis porque:

1. **Pessoas reais são afetadas** — alguém perde o emprego, um relacionamento termina, um time é prejudicado
2. **Você vem evitando a decisão** — o que significa que o problema já está pior do que estava
3. **Irreversibilidade** — ao contrário da maioria das decisões de negócios, você não consegue desfazer isso facilmente
4. **Você tem skin in the game** — seu julgamento sobre a decisão certa está comprometido pelos seus sentimentos sobre ela

Quanto mais tempo você evita uma decisão difícil, pior a situação geralmente fica. A empresa que precisava de um corte de 10% há 6 meses agora precisa de 25%. A conversa com o co-fundador que deveria ter acontecido no mês 4 está acontecendo no mês 14.

**A maioria das decisões difíceis são decisões tardias.**

## O Framework

### Passo 1: O Teste de Reversibilidade

A pergunta mais importante primeiro: **você pode desfazer isso?**

- **Reversível** — tente, aprenda, ajuste (demita o fornecedor, encerre a funcionalidade, mude a estratégia)
- **Parcialmente reversível** — doloroso de desfazer, mas possível (reestruturar, mudar papéis do co-fundador)
- **Irreversível** — não pode ser desfeito (dispensar uma pessoa, encerrar um produto com lock-in de cliente, fechar uma entidade jurídica)

Para decisões irreversíveis, o nível de certeza necessário é maior. Você deve fazer mais due diligence antes de agir. Não porque possa estar errado — mas porque não pode voltar atrás.

**Se você está tratando uma decisão reversível como se fosse irreversível, está evitando-a.**

### Passo 2: O Framework 10/10/10

Faça três perguntas sobre cada opção:

- **10 minutos depois**: Como você vai se sentir imediatamente após tomar esta decisão?
- **10 meses depois**: Qual será o impacto? O problema estará resolvido?
- **10 anos depois**: Quando olhar para trás, esta terá sido a decisão certa?

O sentimento de 10 minutos geralmente é o guia menos confiável. A visão de 10 anos geralmente esclarece qual é a decisão certa.

**A maioria das decisões difíceis parece óbvia em 10 anos. A questão é se você consegue tolerar a dor dos 10 minutos.**

### Passo 3: O Teste Andy Grove

O teste de Andy Grove para decisões: "Se fôssemos substituídos amanhã e um novo CEO assumisse, o que ele faria?"

Um conjunto de olhos frescos, sem investimento emocional no caminho atual, sem custo afundado. Qual é a decisão obviamente certa de fora?

Se a resposta é clara para um estranho, a questão se torna: por que você ainda não fez isso?

### Passo 4: Mapeamento de Impacto nos Stakeholders

Para cada opção, mapeie quem é afetado e como:

| Stakeholder | Impacto Opção A | Impacto Opção B | Reação esperada |
|-------------|----------------|----------------|-----------------|
| Funcionários afetados | | | |
| Time restante | | | |
| Clientes | | | |
| Investidores | | | |
| Você | | | |

Isso não é sobre encontrar a opção que não prejudica ninguém — não existe. É sobre entender o quadro completo antes de decidir.

### Passo 5: O Teste Pré-Anúncio

Antes de tomar a decisão: escreva o anúncio. O e-mail para o time, a mensagem para o cliente, a conversa que você terá.

**Se você não consegue escrever esse anúncio, não está pronto para tomar a decisão.**

Escrever isso força você a confrontar a realidade do que está fazendo. Também evidencia se seu raciocínio se sustenta ao exame. "Estamos fazendo essa mudança porque..." — essa frase soa verdadeira?

### Passo 6: O Plano de Comunicação

Decisões difíceis quase sempre ficam mais difíceis se a comunicação for ruim. A decisão em si não é a única coisa que importa — como ela é feita importa enormemente.

Para cada decisão difícil, planeje:
- **Quem precisa saber primeiro** (a pessoa diretamente afetada, antes de qualquer outro)
- **Como você dirá** (pessoalmente quando possível, nunca por e-mail para impacto pessoal)
- **O que dirá** (honesto, direto, compassivo — veja `references/hard_things.md`)
- **O que eles podem perguntar** (esteja pronto para cada pergunta)
- **O que vem a seguir** (dê a eles um quadro claro do que acontece depois)

## Frameworks Específicos por Tipo de Decisão

### Demitir um Co-Fundador
Veja `references/hard_things.md — Conflitos com Co-Fundadores` para o framework completo.

Perguntas-chave a responder primeiro:
- É um problema de desempenho ou de valores/cultura? (Conversas diferentes)
- Você foi explícito — não apenas insinuou, mas foi direto — sobre o problema?
- Como está a estrutura societária e quais são as implicações jurídicas?
- Há uma função que funciona melhor para eles, ou é uma saída total?
- Quem precisa saber (conselho, time, investidores) e em que ordem?

**A regra:** Se você vem pensando nisso há mais de 3 meses, você já sabe a resposta. A questão é quando, não se.

### Layoffs
Perguntas-chave:
- É um reset único ou o início de um declínio mais longo? (Um reset é recuperável. Layoffs em série destroem a cultura.)
- Você está cortando fundo o suficiente? (Layoffs insuficientes são piores que nenhum — duas rodadas destroem a confiança.)
- Quem é responsável pelo anúncio e ele é direto e honesto?
- Qual é a indenização e ela é justa? (Consulte a CLT para suas obrigações legais.)
- Como você evita que as melhores pessoas saiam depois?

**A regra:** Corte uma vez, corte fundo, corte com dignidade. Incerteza é pior que clareza.

### Pivotando
Perguntas-chave:
- É um pivô verdadeiro (nova direção) ou uma otimização (mesma direção, tática diferente)?
- O que você está mantendo e o que está abandonando?
- Você tem evidências de que a nova direção funciona, ou está fugindo do fracasso?
- Como você conta para os clientes atuais que compraram a visão antiga?
- O que isso faz com a confiança do conselho?

**A regra:** Pivôs devem ser puxados por evidências de nova oportunidade, não empurrados pelo fracasso do caminho atual.

### Encerrando uma Linha de Produto
Perguntas-chave:
- O que acontece com os clientes que atualmente a utilizam?
- Qual é o caminho de migração?
- O que as pessoas que a construíram fazem?
- "Encerrar" é a decisão certa ou "vender" ou "separar" é melhor?
- Qual é a narrativa — interna e externamente?

## O Teste de Estar Evitando

Você sabe que vem evitando uma decisão difícil se:
- Você pensou nela toda semana por mais de um mês
- Você está esperando que a situação "se resolva sozinha"
- Você está aguardando mais dados que você nunca sentirá que são suficientes
- Você já teve a conversa em sua cabeça muitas vezes, mas não na vida real
- Outras pessoas ao seu redor já perceberam o problema

**O custo do atraso é quase sempre maior que o custo da decisão.**

A cada mês que você espera, o problema se acumula. O co-fundador que não está funcionando fica mais entrincheirado. A linha de produto que precisa ser encerrada consome mais recursos. A pessoa que precisa ser desligada afeta as pessoas ao redor dela.

Tome a decisão. Tome-a claramente. Tome-a com dignidade.
